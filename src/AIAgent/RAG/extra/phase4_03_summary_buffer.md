# Phase 4.3：Summary Buffer Memory

## 1. 从「删除」到「压缩」

滑动窗口和 Token Budget Memory 都能防止上下文无限增长，但被淘汰的信息会彻底消失。Summary Buffer 把短期记忆分成两层：

```text
历史摘要（压缩后）
        +
近期原文（完整问答）
        ↓
注入本轮模型上下文
```

新问答会先进入近期原文区。只有近期原文超过 Token 预算时，最早的完整问答才会被批量合并进滚动摘要。

## 2. 摘要保留什么

摘要优先保留：

- 用户的身份、偏好、习惯和明确要求
- 已做出的决定、约束和承诺
- 未完成事项和对后续任务有用的事实
- 新信息对旧信息的更正

寒暄、重复表达和不影响后续的细节会被丢弃。这是有损压缩：它能延长记忆，但不能保证每个细节都存在。

## 3. 两个独立预算

- `max_recent_tokens`：近期完整问答可使用的预算。
- `max_summary_tokens`：滚动摘要可使用的预算。

Phase 4 CLI 使用 DeepSeek V4 官方 tokenizer：近期问答按完整 chat 轮次计数，
摘要按正文计数。摘要调用还会通过 `max_tokens` 要求模型限制输出。
如果模型仍返回过长文本，memory 会对摘要执行硬预算截断，保证它不会无限增长。

## 4. 为什么摘要失败不能拖垮对话

摘要是记忆的辅助流程，Agent 的最终回答在它之前已经生成。如果摘要 API 失败：

1. 旧原文仍然淘汰，确保近期窗口不超预算。
2. 上一版成功摘要保持不变。
3. 错误记录到 `last_summary_error`，但不向上抛出打断主查询。

这会丢失当次被淘汰的信息，但能保证 Agent 仍然可用且上下文有界。

## 5. 运行小预算实验

```bash
uv run python src/AIAgent/RAG/phase4_main.py \
  --strategy summary \
  --token-budget 120 \
  --summary-token-budget 80
```

先告诉 Agent 你的名字、偏好或一个未完成任务，再进行多轮无关对话。使用 `/memory` 观察早期原文被淘汰后，关键信息是否进入历史摘要。

## 6. 局限

- 每次压缩都可能造成细节损失或摘要偏差。
- 摘要仍然是非结构化文本，不适合精确的事实更新与冲突管理。
- 进程退出后会消失，不是长期记忆。
