# AI Agent 技术详解

## 1. 什么是 AI Agent

AI Agent（智能体）是一种能够自主感知环境、做出决策并采取行动来完成目标的 AI 系统。与传统的 LLM 对话不同，Agent 不只是回答问题，它能够主动规划任务、使用工具、与外部环境交互，并根据反馈调整自己的行为。

一个完整的 Agent 通常包含以下核心能力：
- **感知 (Perception)**：理解用户输入和环境状态
- **推理 (Reasoning)**：分析问题，制定行动计划
- **行动 (Action)**：调用工具或执行操作
- **记忆 (Memory)**：记住历史信息和学到的经验
- **反思 (Reflection)**：评估自己的行为，从错误中学习

## 2. Agent 的核心架构模式

### 2.1 ReAct (Reasoning + Acting)

ReAct 是目前最经典的 Agent 架构，由 Yao 等人在 2022 年提出。它的核心思想是让模型交替进行推理（Reasoning）和行动（Acting）。

工作流程：
1. **Thought**：模型思考当前应该做什么
2. **Action**：模型选择一个工具并调用
3. **Observation**：获取工具返回的结果
4. 重复以上步骤，直到得到最终答案

ReAct 的优势在于推理过程是可见的（chain-of-thought），便于调试和理解。

### 2.2 Plan-and-Execute

这种模式先制定完整的执行计划，然后逐步执行：
1. **Planning**：根据用户需求制定包含多个步骤的计划
2. **Execution**：按顺序执行每个步骤
3. **Re-planning**：如果某个步骤失败，重新规划

适合复杂的多步骤任务，但灵活性不如 ReAct。

### 2.3 Reflexion

在 ReAct 基础上增加反思能力：
1. 执行任务
2. 评估执行结果
3. 如果失败，反思失败原因
4. 基于反思经验重新尝试

这种模式能让 Agent 从错误中学习，逐步提升任务完成率。

## 3. 工具使用 (Tool Use)

### 3.1 Function Calling

现代 LLM 大多支持 Function Calling 功能，让模型能够以结构化的方式调用外部工具：
- 定义工具的名称、描述和参数
- 模型根据用户需求选择合适的工具并生成参数
- 系统执行工具调用并返回结果

### 3.2 常见工具类型

- **搜索工具**：网络搜索、知识库检索
- **代码执行**：运行 Python 代码、执行 SQL 查询
- **API 调用**：调用第三方 API（天气、新闻、数据库等）
- **文件操作**：读写文件、处理文档
- **计算工具**：数学计算、数据分析

### 3.3 MCP (Model Context Protocol)

MCP 是 Anthropic 在 2024 年底推出的开放协议，旨在标准化 AI 模型与外部工具/数据源的通信方式。

MCP 的核心概念：
- **Server**：提供工具和资源的服务端
- **Client**：连接 Server 的客户端（通常集成在 Agent 中）
- **Tools**：Server 暴露的可调用功能
- **Resources**：Server 提供的只读数据源
- **Prompts**：Server 定义的 Prompt 模板

MCP 的意义在于：一次编写 Server，所有支持 MCP 的 Agent 都可以使用，实现了工具的标准化和可复用。

## 4. 多 Agent 系统

### 4.1 为什么需要多 Agent

单个 Agent 在处理复杂任务时可能能力不足。多 Agent 系统通过分工协作来解决复杂问题：
- **专业化分工**：每个 Agent 专注于特定领域
- **并行处理**：多个 Agent 同时处理不同子任务
- **交叉验证**：多个 Agent 互相检查结果

### 4.2 多 Agent 协作模式

- **主从模式**：一个主 Agent 负责分配任务，多个从 Agent 执行
- **辩论模式**：多个 Agent 对同一问题提出不同观点，通过讨论达成共识
- **流水线模式**：Agent 按顺序处理，每个 Agent 处理一个阶段
- **层级模式**：类似公司组织架构，层层分解和汇报

### 4.3 常见框架

- **AutoGen (Microsoft)**：支持多 Agent 对话和协作
- **CrewAI**：基于角色的多 Agent 框架
- **LangGraph**：基于图结构的 Agent 编排
- **MetaGPT**：模拟软件公司的多 Agent 系统

## 5. Agent 的记忆系统

### 5.1 短期记忆

即当前对话的上下文窗口。包括用户消息、Agent 的推理过程、工具调用结果等。受限于模型的上下文长度。

管理策略：
- 滑动窗口：只保留最近 N 轮对话
- 摘要压缩：用 LLM 对历史对话做摘要
- Token 截断：超过限制时截断最早的内容

### 5.2 长期记忆

跨会话持久化的信息。通常存储在外部数据库中，需要时通过检索获取。

类型：
- **语义记忆**：存储事实和知识（如用户偏好、学到的信息）
- **情景记忆**：存储过去的经历（如历史任务的执行记录）
- **程序性记忆**：存储如何完成任务的方法和技能

### 5.3 记忆管理

- **存储**：从对话中提取重要信息并持久化
- **检索**：根据当前上下文检索相关的历史信息
- **更新**：当信息发生变化时更新已存储的内容
- **遗忘**：清理不再相关或过时的信息

## 6. Agent 的安全与可控性

### 6.1 常见风险

- **Prompt 注入**：用户通过特殊输入操纵 Agent 的行为
- **工具滥用**：Agent 错误地调用危险工具（如删除文件）
- **无限循环**：Agent 陷入重复推理循环
- **信息泄露**：Agent 泄露系统 Prompt 或敏感信息

### 6.2 安全措施

- **沙箱执行**：在隔离环境中运行工具调用
- **权限控制**：限制 Agent 可以调用的工具和访问的数据
- **循环检测**：设置最大步数或检测重复行为
- **人类监督**：关键操作需要人类确认
- **输出过滤**：检查 Agent 的输出是否包含有害内容
