# Transformer 架构详解

## 1. 概述

Transformer 是 Google 在 2017 年论文《Attention Is All You Need》中提出的一种神经网络架构。它彻底抛弃了传统的循环神经网络（RNN）和卷积神经网络（CNN），完全基于注意力机制（Attention Mechanism）来处理序列数据。

Transformer 的出现是自然语言处理领域的一个里程碑事件。它不仅在机器翻译任务上取得了当时的最佳成绩，还成为了后续几乎所有大语言模型（如 GPT、BERT、LLaMA 等）的基础架构。

## 2. 核心组件

### 2.1 自注意力机制 (Self-Attention)

自注意力机制是 Transformer 最核心的创新。它允许序列中的每个位置都能直接"关注"到序列中其他所有位置的信息，从而有效捕捉长距离依赖关系。

计算过程如下：

1. 对输入向量分别进行线性变换，得到 Query (Q)、Key (K)、Value (V) 三个矩阵
2. 计算注意力得分：Score = Q × K^T / √d_k
3. 对得分进行 Softmax 归一化，得到注意力权重
4. 用注意力权重对 V 进行加权求和，得到输出

其中 d_k 是 Key 向量的维度，除以 √d_k 是为了防止点积值过大导致 Softmax 梯度消失。

### 2.2 多头注意力 (Multi-Head Attention)

多头注意力是自注意力的扩展。它将输入分成多个"头"，每个头独立进行自注意力计算，然后将所有头的输出拼接起来。

多头注意力的好处是让模型能从不同的"角度"关注信息。比如，一个头可能关注语法关系，另一个头可能关注语义关系。

典型配置：GPT-3 使用了 96 个注意力头，每个头的维度是 128。

### 2.3 前馈神经网络 (Feed-Forward Network, FFN)

每个 Transformer 层中，自注意力之后还有一个前馈神经网络。它由两个线性变换和一个非线性激活函数组成：

FFN(x) = W2 · ReLU(W1 · x + b1) + b2

FFN 的作用是对注意力的输出进行非线性变换，增加模型的表达能力。通常中间层的维度是输入维度的 4 倍。

### 2.4 位置编码 (Positional Encoding)

由于 Transformer 没有循环或卷积结构，它本身无法感知输入序列的顺序。因此需要额外添加位置信息。

原始论文使用正弦和余弦函数生成位置编码：
- PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
- PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

后来的模型（如 GPT）改用可学习的位置编码，RoPE（旋转位置编码）等更先进的方案。

### 2.5 残差连接与层归一化

每个子层（注意力、FFN）都使用残差连接（Residual Connection）和层归一化（Layer Normalization）：

output = LayerNorm(x + SubLayer(x))

残差连接帮助解决深层网络的梯度消失问题，层归一化则稳定训练过程。

## 3. 编码器-解码器架构

原始 Transformer 包含编码器（Encoder）和解码器（Decoder）两部分。

### 3.1 编码器

编码器由 N 个相同的层堆叠而成（原始论文 N=6）。每层包含：
1. 多头自注意力机制
2. 前馈神经网络
3. 每个子层后跟残差连接和层归一化

编码器的输出是输入序列的上下文表示。

### 3.2 解码器

解码器也由 N 个相同的层堆叠而成。每层包含：
1. 掩码多头自注意力（Masked Multi-Head Attention）：防止当前位置看到未来位置的信息
2. 编码器-解码器注意力：让解码器关注编码器的输出
3. 前馈神经网络

### 3.3 变体

后续模型根据需要选择不同的架构：
- **仅编码器**（如 BERT）：适合理解任务（分类、NER 等）
- **仅解码器**（如 GPT 系列、LLaMA）：适合生成任务
- **编码器-解码器**（如 T5、BART）：适合序列到序列任务（翻译、摘要等）

## 4. 训练策略

### 4.1 预训练 (Pre-training)

大语言模型通常先在大规模无标注文本上进行预训练：
- **自回归语言模型**（如 GPT）：预测下一个 token
- **掩码语言模型**（如 BERT）：预测被遮盖的 token

### 4.2 微调 (Fine-tuning)

预训练后，可以在特定任务的数据上进行微调：
- 全量微调 (Full Fine-tuning)：更新所有参数
- 参数高效微调 (PEFT)：只更新少量参数，如 LoRA、Adapter 等

### 4.3 对齐训练

为了让模型更好地遵循人类指令：
- SFT (Supervised Fine-Tuning)：在指令-回答对上训练
- RLHF (Reinforcement Learning from Human Feedback)：通过人类反馈强化学习
- DPO (Direct Preference Optimization)：直接偏好优化，简化 RLHF 流程

## 5. 关键的 Scaling Laws

Kaplan 等人发现，Transformer 模型的性能（loss）可以用幂律关系来预测：

L(N) ∝ N^(-0.076)  （N 是参数量）
L(D) ∝ D^(-0.095)  （D 是数据量）
L(C) ∝ C^(-0.050)  （C 是计算量）

这意味着增大模型、数据或计算量，性能会持续提升，且这种关系在很大范围内是可预测的。这也是"大力出奇迹"路线的理论依据。
