1. 数据结构：`State`（状态 = 图里的节点）、`Frag`（片段 = 一个带着悬空线头的盒子）。← 今天

3. 解析编译：正则串 → 后缀表达式 → 用一个栈把 `Frag` 拼成完整 NFA。

## 第一块砖：怎么用代码表示"状态"？

你前置知识里说"有限的比特决定了状态的有限性"——那么在代码里， **一个状态就是图里的一个节点** ，整台机器就是这些节点连成的有向图。关键问题是：一个节点需要带多少信息？

Thompson 给了一个近乎极简的答案：**每个状态最多只有两条出边，且只有三种形态。**

```
  字符态 (char)        分裂态 (split)            接受态 (match)
                          .---> out
   --[a]--> out      --( ε )                    --[ OK ]   （没有出边）
                          '---> out1
```

* **字符态** ：守着一个具体字符 `a`。"吃掉"这个字符后，沿唯一一条出边 `out` 走。这就是你说的串联里那道"门"。
* **分裂态** ：不消耗任何字符（ε 转移），但同时分叉成两条路 `out` / `out1`。这就是 Kleene 里"并联"和"自环/零次捷径"的物理实现——所有的分支，本质上都是它。
* **接受态** ：终点，没有出边，光标走到这里就算匹配成功。

为什么"最多两条"就够了？因为 `a|b` 是二选一（分叉成 2），`*` 是"进循环体 or 跳过"（也分叉成 2）。任何 n 路分支都能用嵌套的二路分裂搭出来——这正是 McCulloch-Pitts 的精神：用最简单的二元单元堆出任意复杂的逻辑。

落成代码就这么几行：

```python
# 两个"非字符"的标签，用负数当哨兵，绝不会和真实字符撞上
SPLIT = -1   # 分裂态：不吃字符，同时走两条 ε 路
MATCH = -2   # 接受态：到这儿就匹配成功

class State:
    def __init__(self, c, out=None, out1=None):
        self.c    = c       # 守门的字符；或 SPLIT / MATCH
        self.out  = out      # 第一条出边（指向另一个 State）
        self.out1 = out1     # 第二条出边（只有 SPLIT 用得到）
```

注意 `out` 初始是 `None`——这就是下一块砖的引子。

## 第二块砖：怎么表示"还没接上的线头"？

一个孤零零的 `State` 是死的。当我为 `a` 造好节点时，它的 `out` 该指向谁？ **我还不知道** ——后面跟的是 `b` 还是 `c` 还是结尾，要等外层表达式才知道。所以这条出边此刻是 **悬空的** ，像一根焊好一头、另一头还在空中晃的电线。

Thompson 构造法的灵魂数据结构就是为此而生的—— **`Frag`（片段）= 一个入口 + 一把悬空的线头** ：

```
            +-------------------+
   入口 ---> |   某个子表达式     | ===> 线头1（悬空）
   start    |   编译出的小图      | ===> 线头2（悬空）
            +-------------------+
```

每个子表达式都编译成这样一个盒子：有一个确定的"入口状态"，背后伸出若干根还没接的线。要把两个盒子串联，就是把前一个盒子的所有线头，焊到后一个盒子的入口上——这就是你说的"符号侧操作 ↔ 物理接线"在代码里的样子。

那"一根悬空线头"在 Python 里怎么存？一条出边无非是 **某个 `State` 的 `out` 或 `out1` 字段** 。我们没法像 C 那样存一个指向该字段的指针（`**out`，那是原版最漂亮的地方），所以退而求其次：用一个二元组 `(state, 字段名)` 记下"是哪个节点的哪根线"。将来焊接时，`setattr(state, 字段名, 目标)` 就接上了。

```python
class Frag:
    def __init__(self, start, outs):
        self.start = start    # 这个片段的入口状态
        self.outs  = outs      # 悬空线头列表，每个元素是 (state, 'out' 或 'out1')
```

## 验收：最小的那个片段

把两块砖一拼，我们就能造出整个构造法的 **基本情形** ——单个字符 `a` 的片段。它小到只有一个节点、一根悬空线：

```python
def literal(c):
    s = State(c)                  # 造一个守门 c 的字符态，out 先悬空
    return Frag(s, [(s, 'out')])   # 入口就是它自己，唯一的线头是它的 out
```

```
            +---+
   start --> | a | ===> (out 悬空，等着外层来焊)
            +---+
```

这短短两行其实把整个 Thompson 构造的套路浓缩了： **造节点 → 返回一个 Frag，把悬空的线头都暴露出来** 。后面三个组合子，无非是"接收别人的 Frag，造几个新节点，重新焊线，再返回一个新 Frag"而已。

---

数据结构这层就到这。下一步是真正有意思的部分：把 Kleene 的三种操作——连接（串联线头）、选择（造一个分裂态做并联）、重复（造分裂态做自环 + 零次捷径）——写成三个焊接函数。
